	Одна или более джоб могут быть организованы в одну или более стадий (Stages)
	Роль Jobs в выполнении Плана Bamboo:
		► Bamboo Job - отдельная единица сборки в плане;
		► джоба позволяет собрать вместе несколько задач (тасков), которые мы хотели бы выполнить последовательно

__Настройка требований заданий (job's requirements)__
Требования указываются:
	• в джобах
	• в тасках

Требование определяет возможность, которой должен обладать агент для создания этой джобы или таска.
Джоба наследует все требования, указанные в её тасках.
Каждая джоба может быть построена только агентами, чьи возможности соответствуют требованиям джобы.

В Bamboo есть четыре типа возможностей, которые можно указать в соответствии с требованиями работы и задачи:
	1. Возможности исполняемых файлов (Executable capabilities)
	   — определение внешних программ, которые может вызывать Bamboo,
	   например Ant, Maven, MSBuild или PHPUnit
	2. Возможности JDK (JDK capabilities)
	   — определите версии JDK, которые будут использоваться джобой или таском.
    3. Возможности управления версиями (Version control capabilities )
	   — укажите клиентское приложение VCS, которое Bamboo должен использовать для проверки исходного кода.
    4. Пользовательские возможности (Custom capabilities)
	   — могут использоваться для управления тем, какие джобы будут создаваться конкретным агентом.
	   Например, если сборки для определенного задания должны выполняться только в среде Windows,
	   вы можете создать пользовательскую возможность «operating.system=WindowsXP» для соответствующего(их) агента(ов)
	   и указать это как требование для этого задания.

Прежде чем Вы сможете определить требование в Вашей джобе, сначала Вы должны определить возможность в Вашей системе Bamboo

__Указание дополнительных требований к работе__

Джоба будет наследовать требования от своих тасков по умолчанию.
Однако Вы можете указать дополнительные требования для джобы, в дополнение к требованиям её тасков.

Чтобы указать дополнительные требования к джобе:
	1. Перейдите на страницы конфигураций желаемой джобы (как описано в https://confluence.atlassian.com/bamboo0607/configuring-jobs-966045090.html)
	
	2. Перейдите на вкладку «Требования» (Requirements).
	   На этой странице показан список всех текущих требований джобы,
	   а также количество «совпадающих агентов»  ('Matching Agents') и «совпадающих образов» ('Matching Images')
	   (т. е. агентов/эластичных образов, которые соответствуют требованиям задания и могут запускать сборку для этого задания).
	   
	3. Если вы ранее настроили возможность агента, вы можете выбрать ее из списка требований
       в разделе «Добавить дополнительное требование».
	   Если вы настраиваете новое настраиваемое требование, вместо этого выберите Новое настраиваемое требование из этого списка.
	   
	4. Заполните форму требования:
		a. Ключ (только для новых пользовательских требований) - введите ключ новой способности
		b.Выберите значение для требования из списка:
		    exists (существует) — эта джоба может быть построена любым агентом, который имеет способность (capability) с тем же ключом.
            equals (равно) — это задание может быть построено любым агентом, который имеет способности с тем же ключом и значением.
            matches (соответствует) — это задание может быть построено любым агентом, у которого есть способности с тем же ключом,
                    и значение соответствует регулярному выражению.

Щелкните "Добавить" → Количество «соответствующих агентов» и «совпадающих образов» будет обновлено,
так как план теперь может быть создан только агентами с возможностями, которые соответствуют новым указанным вами настраиваемым требованиям.
