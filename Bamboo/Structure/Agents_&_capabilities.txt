‼ Можно запустить несколько агентов Bamboo на одной и той же машине.
  Для этого нужно лишь предоставить раздельные домашние каталоги для установки каждого агента.

Агент может запустить задание (джобу), если его возможности (capabilities) совпадают с требованиями (requirements) джобы.
Каждое задание (job) наследует требования от отдельных задач (tasks), которые оно содержит.
► Возможности, которые можно определить для агента ([3]→): ◄
	• исполняемый файл (напр., Maven)
	• JDK (Java Development Kit)
	• клиентское приложение системы контроля версий (напр., Git)
	• пользовательская возможность (custom capability)
	  [ это свойство типа "ключ-значение", определяющее конкретную характеристику агента
	    (например, "operating.system=WindowsXP" или "fast.builds=true")
	  ]

Возможности обычно:
	♦ определяют путь к УЖЕ УСТАНОВЛЕННОМУ исполняемому файлу,
	♦ должны быть определены в Bamboo, прежде чем Bamboo или его агенты смогут их использовать.

Возможности могут быть определены специально для агента или могут быть общими для всех локальных/удаленных агентов. 
‼ Значение специфичной для агента возможности ПЕРЕОПРЕДЕЛЯЕТ значение общей возможности с тем же именем (если она существует).

Как возмжности взаимодействуют с требованиями
	Требование указывается в джобе или таске.
	Требование определяет возможность, которой должен обладать агент для выполнения конкретной джобы или таска.
	Джоба наследует все требования, указанные в его тасках.
	Вместе возмжности и требования определяют, какой агент может выполнить билды для конкретных Jobs.
	

Как сборки распространяются среди агентов ([3]→):
	Агент будет выполнять одно задание (job) за раз и блокировать любые другие задания Bamboo,
	пока не будет завершена сборка этого задания.
	Если вы хотите создать несколько заданий (jobs) ОДНОВРЕМЕННО на сервере Bamboo, просто настройте несколько локальных агентов.
	Если агенты являются удаленными, вам потребуется установить такое же количество экземпляров агентов на машине.
	Требуются отдельные установки, поскольку каждому удаленному агенту потребуются собственные домашние каталоги и каталоги журналов.

Изменение и удаление возможностей

В зависимости от типа возможности, вы можете редактировать такие параметры capabilities, как
	• путь (Path);
	• домашний директорий Java (Java Home);
	• значение (Value)
	
Что следует иметь в виду:
  1. поскольку каждый агент может запустить лишь те сборки для джоб, чьи требования соответствуют возможностям агента,
     изменение или удаление возможности может означать, что некоторые планы больше не смогут быть построены
  2. Переименование возможности включает изменение её ключа

Для изменения возможности конкретного агента (agent-specific) необходимо перейти в настройки этого агента.
Возможности локального сервера и общие удалённые возможности настраиваются в меню:
	* → Overview → [Server Capabilities/Shared Remote Capabilities]
