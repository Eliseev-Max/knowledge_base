Настройка связанного репозитория для Bamboo Specs

Bamboo Specs требуют настройки связанного репозитория.
Перейдите в "Настройки" → "Связанные репозитории" (для этого потребуются права администратора),
чтобы создать или обновить конфигурацию связанного репозитория.
Два наиболее важных шага, которые необходимо предпринять здесь, заключаются в следующем:

    1. Определите, какая ветвь вашего репозитория будет использоваться Bamboo для чтения файла спецификаций.
	   Источником истины можно считать только одну ветку;
	   [ личная рекомендация автора: сделать ее веткой разработки ]
    2. Чтобы Bamboo Specs могли выполнять свою работу, необходимо правильно настроить два набора разрешений:
		 а. Во-первых, проект Bamboo должен предоставить разрешения связанному_репозиторию для создания и изменения планов сборки и развертывания.
	        Перейдите к проекту, в котором вы хотите, чтобы ваш связанный_репозиторий работал:
			► «Настройки проекта» → «Репозитории Bamboo Specs» → добавить связанный репозиторий.

		б. На вкладке Bamboo Specs связанного репозитория включите Доступ ко всем проектам и Доступ ко всем репозиториям.
	       Я совершенно уверен, что это требование не является передовой практикой безопасности, но,
	       по моему опыту, Bamboo Specs не будет работать без них.

Тестирование Bamboo Specs
	• Создайте тестовую ветку (feature/bugfix ветку);
    • Измените имена и ключи планов сборки и развертывания и отправьте их в источник,
	  чтобы ваши изменения не перезаписали существующие планы;
    • Установите ветку тестирования в качестве ветки связанного репозитория по умолчанию на вкладке «Общие» и продолжите внесение изменений.
    • Как только вы довольны изменениями и они были проверены: установите ветку связанного репозитория обратно в ветку разработки,
	  затем верните имена планов и ключи в ветке тестирования на существующие планы.
    • Удалите планы тестирования вручную.
	
Неожиданные исключения Bamboo Specs
	- невозможность установить зависимости плана
	!!! использовать артефакты Bamboo исключительно для планов отладки сборки и развертывания, 
	!!! а также подходящие репозитории пакетов для хранения и извлечения артефактов версии сборки
	
	[ При экспорте спецификаций Bamboo из существующих планов планы сборки и планы развертывания не считаются сильно связанными,
	  поэтому вам потребуется собрать и объединить спецификации из обоих файлов в один.
	  Для этого скопируйте спецификации плана развертывания под спецификациями плана сборки, сохранив --- в качестве разделителей:
	  ---
	  спецификации_плана_сборки
	  ---
	  спецификации_плана_развертывания
	  ]

ПРИМЕЧАНИЕ: порядок разделов важен!
	За определением плана сборки должны следовать 
	► разрешения плана сборки,
	► затем определение(я) плана развертывания, причем за каждым определением плана развертывания следует раздел для его разрешений.
