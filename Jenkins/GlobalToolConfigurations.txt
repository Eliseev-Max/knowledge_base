► На серверах Linux мы можем установить только одну версию инструмента одновременно,
(например:
  мы можем установить только версию maven 3.6 
  и не можем установить обе версии maven 3.6 и 3.7 на одном сервере.
)

Настроив глобальную конфигурацию инструментов (Global Tool Configurations),
мы можем установить несколько версий maven, nodejs, gradle, java и ant (и других tools)

Разберёмся с версионированием глобальных инструментов на примере Maven.

  1. Jenkins Dashboard → Manage Jenkins
  2. Кликнуть по "Global Tool Configurations"
  3. Кликнуть по "Maven installations"
  4. Прокрутить вниз и нажать на "Add Maven"
  5. Задать уникальное имя в поле "Name" и выбрать подходящую версию maven из выпадающего меню
  6. Кликнуть по "Add Maven"
  7. Повторить шаги 5-6.


► Как использовать maven требуемой версии в Jenkinsfile
---
tools {
    maven 'maven-3.6.3'
}
---
# maven-3.6.3 - это уникальное имя, которое мы задали в Global Tool Configurations.
  Когда мы используем maven В ПЕРВЫЙ РАЗ, ссылаясь на это имя,
  и запускаем задание, jenkins:
    • автоматически скачает tar-файл maven,
    • извлечет его и сохранит в папке /var/lib/jenkins/tools/

[!] Начиная со следующей сборки, Jenkins будет использовать maven из этой папки,
    инструмент не будет загружаться каждый раз.

  Configure System — раздел, где администратор может определить глобальные настройки,
  которые применяются КО ВСЕМ проектам.
  
► Manage Jenkins → Configure System
::(страница глобальных конфигураций)

########################################
# Некоторые возможности этой страницы: #
########################################

Jenkins home directory
  - сообщает о размещении домашнего каталога Jenkins в системе.

System Message
  - отображает сообщения на странице Jenkinsdashboard.

Executors
  - сообщает, сколько параллельных джоб можно запустить.

Usage option
  - говорит о том, как графики Jenkins строятся на любом узле.

Quiet period
  - если этот параметр не равен нулю, новые сборки проекта добавятся в очередь, но Jenkins будет ждать указанный период времени (в секундах) перед фактическим запуском.

Jenkins URL
  - по умолчанию отображает HTTP-адрес установки Jenkins в форме localhost.

Environment variable
  - переменные среды применяются для каждой сборки на каждом узле. Их можно использовать в конфигурации Jenkins (*as $key or ${key} *), и они добавятся в среду процессов, запущенных из сборки.

Usage statistics
  - помогает отслеживать данные об использовании Jenkins.

Git Plugin
  - применяется для установки глобального имени пользователя git и данных электронной почты в Jenkins.

Email notification
  - используется для настройки конфигураций, связанных с электронной почтой, для отправки результатов после запуска сборок.
