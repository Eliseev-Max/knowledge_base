
Порт (в компьютерных сетях) - это номер, присвоенный для однозначной идентификации конечной точки соединения и передачи данных в определенную службу.
Порт (на программном уровне, в операционной системе) - это логическая конструкция, которая идентифицирует конкретный процесс или тип сетевой службы.
Порт идентифицируется для каждого транспортного протокола и комбинации адресов 16-разрядным числом без знака, известным как номер порта.
Наиболее распространенными транспортными протоколами, использующими номера портов, являются Протокол управления передачей (TCP) и протокол пользовательских дейтаграмм (UDP). 

Порт может находиться в одном из следующих состояний:
	• открыт (open);
	• фильтруется (filtered);
	• закрыт (closed);
	• без фильтрации (unfiltered)

Порт называется открытым, если какое-либо приложение, служба на целевой машине прослушивает (listening) его в ожидании соединений/пакетов.
Также по отношению к службе часто употребляют термин «привязка» (bind).
Привязка службы к порту = служба назначена к определённому порту (после запуска службы, она будет прослушивать порт, чтобы ответить на входящие запросы).
Термин «привязка» используют по отношению к службам, которые прослушивают порт на постоянной основе и готовы принять входящие соединения.
Примеры таких служб:
	► SSH
	► DB-Server (MySQL, PostgreSQL)
	► Web-сервер.

Некоторые приложения (например, веб-браузер) делают исходящие соединения.
Для этих исходящих соединений обычно используется порт с номером в десятки тысяч, например, 37830 или 46392 или 54284 или что-то подобное.

‼ Но они используют порт на короткое время — только чтобы отправить запрос и получить ответ.
Сразу после получения ответа программа освобождает порт.
Если ей понадобится вновь сделать запрос, то она может выбрать любой другой незанятый порт — необязательно тот же самый.

Такие порты не являются прослушивающими, то есть они не принимают внешние запросы на инициализацию соединения.
Это состояние сокета называется ESTABLISHED, его нужно отличать от прослушивания порта (LISTENING), то есть от открытых портов.
	► ESTABLISHED != LISTENING ◄

Способы определения прослушиваемых портов в системе:

1. Используя команду netstat (считается устаревшей)
	$ sudo netstat -lntup
	# -l → вывести все прослушивающие сокеты;
	# -t → показать все TCP-соединения;
	# -u → показать все UDP-соединения;
	# -p → показывать PID и имя программы/приложения, прослушивающего порт;
	# -n → отображать порты как числа (без этого флага будут выведены имена популярных служб).

	Определить, какое приложение прослушивает определённый порт:
		$ sudo netstat -lntup | grep "httpd"		# по имени службы
		$ sudo netstat -lntup | grep ":80"			# по номеру порта
	
	Для ежесекундного обновления данных использовать флаг: -c
	
2. Используя команду ss
	(отображение информации о сокетах)
	
	Показать все прослушиваемые порты для TCP и UDP соединений в виде цифровых значений:
	  $ sudo ss -lntu
	# -l → показать только прослушивающие сокеты (по умолчанию они опускаются);
	# -t → показать TCP сокеты;
	# -u → показать UDP сокеты;
	# -p → показать процессы, использующие сокет.
	
3. Используя программу Nmap
	Nmap — это мощный и популярный инструмент исследования сети и сканер портов.
	Пакет необходимо устанавливать в систему:
	  $ sudo apt install nmap
	
	Сканирование открытых/прослушиваемых портов в системе:
		$ sudo nmap -n -Pn -sS -sU -p- localhost

4. Используя команду lsof
	(вывода списка открытых файлов (= потоков | сетевых файлов) в Linux)
	
	Вывода списка всех Интернет и сетевых файлов
	  $ sudo lsof -i
	
	Найти приложение, прослушивающее определённый порт (например, 80):
	  $ sudo lsof -i :80
