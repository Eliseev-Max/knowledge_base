Predefined Values

Значения, заданные в файле values.yaml (или с помощью флага --set), доступны из объекта .Values в шаблоне.
Но есть и другие предопределенные части данных, к которым вы можете обращаться в своих шаблонах.

Следующие значения
  -> являются предопределенными,
  -> доступны для каждого шаблона и
  -> не могут быть переопределены.

Как и для всех values, ИМЕНА ЧУВСТВИТЕЛЬНЫ К РЕГИСТРУ.

  ● Release.Name:
      = Название релиза (! не чарта! )

  ● Release.Namespace:
      = namespace, в котором был запущен/выпущен чарт

  ● Release.Service:
      Service, который провёл релиз. В Helm это всегда Helm!
      # см. "Built-in Objects": https://helm.sh/docs/chart_template_guide/builtin_objects/

  ● Release.IsUpgrade:
      = принимает значение true, если текущая операция является upgrade или rollback

  ● Release.IsInstall:
      = принимает значение true, если текущая операция является install.

  ● Chart:
      = Содержимое файла Chart.yaml.
        Таким образом, версию графика можно получить в Chart.Version,
        а сопровождающих (maintainers) - в Chart.Maintainers.

  ● Files:
      = map-подобный объект, содержащий все неспециальные файлы в чарте.
        Это не даст вам доступа к шаблонам, но даст вам доступ к дополнительным файлам, которые присутствуют
        (если они не исключены с помощью .helmignore).
        Доступ к файлам можно получить с помощью:
        {{ index .Files «file.name» }}
        или с помощью функции:
        {{.Files.Get name }}.
        Вы также можете получить доступ к содержимому файла в виде []byte (байта), используя:
        {{ .Files.GetBytes }}

  ● Capabilities:
      = map-подобный объект, содержащий информацию
        - о версиях Kubernetes ({{ .Capabilities.KubeVersion }})
        - и поддерживаемых версиях Kubernetes API ({{ .Capabilities.APIVersions.Has «batch/v1» }}).


[!] Все неизвестные поля Chart.yaml будут удалены.
    Они НЕ БУДУТ ДОСТУПНЫ внутри объекта Chart.
    Таким образом, Chart.yaml нельзя использовать для передачи произвольно структурированных данных в шаблон.
    Однако для этого можно использовать файл values.
