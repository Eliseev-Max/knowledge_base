NTLM - это схема, определенная компанией Microsoft.
Это более безопасная схема, чем Basic, но менее безопасная, чем Digest.
NTLM можно использовать с прокси-серверами или серверами, НО не с обоими одновременно.
Если используется прокси-сервер, то он не может быть использован для аутентификации сервера.
Это связано с тем, что протокол фактически проверяет подлинность TCP-соединения, а не отдельных HTTP-взаимодействий.

На платформах Microsoft Windows аутентификация NTLM пытается получить учетные данные пользователя от системы, не запрашивая объект аутентификатора пользователя (аутентификатор = окно с полями ввода логина и пароля).
Если эти учетные данные не будут приняты сервером, то будет вызван аутентификатор пользователя.

Поскольку класс Authenticator был определен до появления поддержки NTLM, не было возможности добавить в API поддержку поля NTLM domain.
Существует три варианта указания домена:
	1. Не указывать его.
		В некоторых средах домен фактически не требуется, и приложению не нужно его указывать.
    2. Имя домена может быть закодировано в имени пользователя путем префиксации имени домена с обратной косой чертой '\' перед именем пользователя.
		При использовании этого метода существующие приложения, использующие класс Authenticator, не нуждаются в модификации, если только пользователи будут предупреждены о необходимости использования этой нотации.
    3. Если имя домена не указано, как в методе (2), и определено системное свойство "http.auth.ntlm.domain", то в качестве имени домена будет использоваться значение этого свойства.
