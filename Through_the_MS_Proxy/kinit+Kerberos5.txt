Typically, clients find Kerberos servers for their default realm in the domain-name system.
Servers for your realm were found in DNS.
For most configurations it is best to use DNS to find these servers so that if the set of servers for your realm changes, you need not reconfigure each machine in the realm.
However, in special situations, you can locally configure the set of servers for your Kerberos realm.
Add locations of default Kerberos servers to /etc/krb5.conf?
===
Обычно клиенты находят серверы Kerberos для своего пространства по умолчанию в системе доменных имен.
Серверы для вашего царства были найдены в DNS.
Для большинства конфигураций лучше всего использовать DNS для поиска этих серверов, чтобы, если набор серверов для вашего царства изменится, вам не нужно было заново настраивать каждую машину в царстве.
Однако в особых ситуациях вы можете локально настроить набор серверов для вашего царства Kerberos.
Добавить расположение серверов Kerberos по умолчанию в файл /etc/krb5.conf?
