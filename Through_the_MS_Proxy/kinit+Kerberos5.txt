===
Обычно клиенты находят серверы Kerberos для своего пространства по умолчанию (default realm) в системе доменных имен (DNS).
Серверы для вашей среды были найдены в DNS.
Для большинства конфигураций лучше всего использовать DNS для поиска этих серверов, чтобы, если набор серверов для вашего пространства изменится, вам не нужно было заново настраивать каждую машину в царстве.
Однако в особых ситуациях вы можете локально настроить набор серверов для вашего пространства Kerberos.
Добавить расположение серверов Kerberos по умолчанию в файл /etc/krb5.conf?
---
Перво-наперво
Kerberos - один из самых надежных протоколов аутентификации для большинства рабочих сред.
Он обеспечивает надежную однократную регистрацию или вход в сеть для пользователей в незащищенных сетях.
В идеале Kerberos предоставляет пользователям билеты (тикеты, tickets), чтобы помочь им минимизировать частое использование паролей в сетях.
Частое использование паролей = риск утечки учётных данных, кражи пароля.


Основные понятия
  ♦ Key Distribution Center (KDC) - хранилище информации о паролях пользователей.
  ♦ Admin server - основной сервер kerberos.
  ♦ Realm - "среда", в которой производится аутентификация.
  ♦ Principal - пользователь или сервис, участвующий в механизме аутентификации. 

Центральной частью схемы аутентификации Kerberos является третья доверенная сторона - Key Distribution Center (KDC),
которая является централизованным хранилищем информации о пользователях.
Перед разворачиванием Kerberos, должен быть выбран сервер, который будет выполнять роль KDC.

(!) Физическая и сетевая безопасность критичны для этого сервера, так как его компрометация ведет к компрометации всего realm
(!) Важно выбрать хорошее имя для realm. По правилам, имя realm это доменное имя сайта в верхнем регистре.

Требования при настройке realm:
  * все серверы и клиенты, которые входят в realm Kerberos должны иметь возможность взаимодействовать между собой;
  * время между устройствами в realm должно быть синхронизовано. 

Имена хостов
‼ Каждый сервер внутри Kerberos realm должен иметь Fully Qualified Domain Name (FQDN).
+ Kerberos так же ожидает, что FQDN сервера является reverse-resolvable.

Если выяснение доменного имени по IP недоступно, то установите значение переменной rdns = false на клиентах в файле krb5.conf.

Проверка корректности обнаружения forward и reverse:
  $ nslookup server.example.com   # -> > IP-адрес сервера
  $ nslookup <server ip address>  # -> > FQDN сервера
