в статье:
  * описана установка Sonatype Nexus в контейнере Docker с использованием Nginx в качестве обратного прокси сервера,
  * рассмотрено получение бесплатного доменного имени на сайте freenom.com
  * создание SSL-сертификата при помощи службы Let’s Encrypt.


# Считается хорошей практикой сохранять данные Nexus в директории на хост-машине,
# которую в процессе запуска мы примапим к контейнеру nexus.

---<вариант запуска одиночного контейнера nexus>---
docker run -d -p 8081:8081 --name nexus -v nexus-data:/nexus-data sonatype/nexus3
---

# Далее создаётся/регистрируется домен с помощью сервиса Freenom
# вводится имя желаемого домена, выбирается зона...
# также полезные опции:
## + переадресация URL
## + предоставление бесплатного DNS-сервиса

# После создания домена следует процедура добавления DNS-записей типа A и TXT.


Следующий шаг после установки и настройки домена:
  => Установка certbot и let’s encrypt

Certbot
  — это ACME-клиент, который автоматически создаёт и устанавливает сертификаты,
    не требуя остановки или перезапуска web-сервера.

# Данный клиент рекомендует использовать сам let’s encrypt.
---
sudo apt -y install certbot letsencrypt
---

---[COMMAND]---
sudo certbot certonly --manual --agree-tos --email <email-address> --preferred-challenges=dns -d <имя_домена>
---

Завершительный этап:
  = Настройка Nginx в качестве reverse proxy

#[!] Советую выставлять proxy_buffer off в nginx,
#### иначе могут быть неожиданные обрывы скачивания крупных >1 гб артефактов.
