################
# Nexus + LDAP #
################
(https://help.sonatype.com/en/ldap.html)

Nexus Repository Manager может использовать Lightweight Directory Access Protocol (LDAP)
для аутентификации через внешние системы, обеспечивающие поддержку LDAP, такие как:
  * Microsoft Exchange/Active Directory,
  * OpenLDAP,
  * ApacheDS
  и другие.

Настройка LDAP может быть выполнена в несколько простых шагов:

  1. Включить LDAP Authentication Realm
  2. Создать конфигурацию сервера LDAP с соединениями и деталями отображения пользователей/групп
  3. Создать внешние сопоставления ролей для адаптации ролей LDAP к специфике использования менеджера репозитория

В дополнение к обработке аутентификации:
  => менеджер репозиториев можно настроить на сопоставление ролей с группами пользователей LDAP.
  Если пользователь является членом группы LDAP, которая соответствует ID роли,
  менеджер хранилищ предоставляет ему соответствующую роль.
  
  В дополнение к этой настраиваемой возможности сопоставления пользователей и групп
  менеджер хранилищ может дополнить членство в группах LDAP специфическим сопоставлением пользователей и ролей.

Менеджер репозитория может кэшировать информацию об аутентификации и поддерживает несколько LDAP-серверов и сопоставлений пользователь/группа.

непосредственно из пользовательского интерфейса можно проверить
  + детали подключения к LDAP-серверу и сопоставления пользователей/групп, 
  + конкретные логины учетных записей

---
Все эти возможности позволяют
  -> адаптироваться к любому конкретному сценарию использования LDAP
  -> использовать преимущества централизованной аутентификации, установленной в вашей организации ВО ВСЕХ менеджерах репозиториев.


***************************************
* Включение LDAP Authentication Realm *
***************************************

Активируйте LDAP Realm, выполнив следующие действия:

  1. Перейдите в раздел администрирования Realms
  2. Выберите LDAP Realm и добавьте его в список Active Realms справа.
  3. Убедитесь, что LDAP Realm находится под Local Authenticating Realm в списке
  4. Нажмите Сохранить

[!] Лучше всего оставить Local Authenticating Realm АКТИВИРОВАННЫМ,
    чтобы менеджер репозиториев мог использоваться
      - анонимными пользователями,
      - администраторами
      - и другими пользователями, настроенными в этом царстве,
      даже если аутентификация LDAP отключена или недоступна.

[!] Любая учетная запись пользователя, не найденная в Local Authenticating Realm,
    будет передана для аутентификации LDAP.
