Блок map из модуля ngx_http_map_module
#(https://nginx.org/en/docs/http/ngx_http_map_module.html)
  Модуль ngx_http_map_module создает переменные, значения которых зависят от значений других переменных.
---
Syntax:	map string $variable { ... }
Default:	—
Context:	http
---
  Создает новую переменную, значение которой зависит от значений одной или нескольких исходных переменных, указанных в первом параметре.

(!)  Поскольку переменные оцениваются только при их использовании,
  простое объявление даже большого количества "map"-переменных не добавляет никаких дополнительных затрат на обработку запроса.

  Параметры внутри блока map задают соответствие между исходными и результирующими значениями.
  Исходные значения задаются в виде строк или регулярных выражений (0.9.6).
  Строки сопоставляются БЕЗ УЧЕТА РЕГИСТРА.
  Регулярное выражение:
    должно начинаться:
      - для сопоставления с учетом регистра: с символа `~`
      - для сопоставления без учета регистра: с символов `~*` 

   может содержать именованные и позиционные захваченные значения,
     которые в дальнейшем могут быть использованы в других директивах вместе с результирующей переменной.
     Если исходное значение соответствует одному из имен специальных параметров, описанных ниже, оно должно быть снабжено префиксом "\".

  Результирующее значение может содержать:
    - текст,
    - переменную (0.9.0)
    - и их комбинацию (1.11.0).

Также поддерживаются следующие специальные параметры:
-->
  default value
<--
  устанавливает результирующее значение, если исходное значение не соответствует ни одному из указанных вариантов. 
  Если значение default не указано, результирующим значением по умолчанию будет пустая строка.

hostnames
  = указывает, что исходными значениями могут быть имена хостов с префиксной или суффиксной маской:

*.example.com 1;
example.*     1;

  Следующие две записи:
---
example.com   1;
*.example.com 1;
---
  можно комбинировать:
---
.example.com  1;
---

Этот параметр должен быть указан перед списком значений.

include file
  includes a file with values. There can be several inclusions.
