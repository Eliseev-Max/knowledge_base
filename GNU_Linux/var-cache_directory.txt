Что содердится в директории /var/cache ?
Можно ли удалять содержимое этой директории, не опасаясь нанести вред ОС?

[source:]
  1) https://unix.stackexchange.com/questions/23156/can-i-safely-remove-var-cache
  2) "5.5. /var/cache : Application cache data": https://refspecs.linuxfoundation.org/FHS_3.0/fhs/ch05s05.html
  3) "Стандарт на структуру каталогов файловой системы. (Filesystem Hierarchy Standard).
      5.5 /var/cache : Данные кэша приложений"
      -> https://www.opennet.ru/docs/RUS/file-sys/fhs-5.5.html

  Каталог /var/cache содержит кэшированные файлы,
  то есть файлы, которые были сгенерированы и могут быть повторно сгенерированы в любое время;
  НО их стоит сохранить, чтобы сэкономить время на их повторном вычислении.

  ЛЮБОЕ ПРИЛОЖЕНИЕ может создать здесь файл или каталог.
  Предполагается, что хранящиеся здесь файлы НЕ ЯВЛЯЮТСЯ КРИТИЧЕСКИМИ,
  поэтому СИСТЕМА МОЖЕТ УДАЛЯТЬ содержимое /var/cache
    -> либо периодически,
    -> либо когда его содержимое становится слишком большим.

  Любое приложение должно учитывать, что хранящийся здесь файл может исчезнуть в любой момент,
  и быть готовым пересчитать (повторно выяислить) его содержимое (с некоторым временным штрафом).


#################################################
# Почему НЕ СТОИТ удалять содержимое /var/cache #
#################################################

[мнение:] /var/cache/bind/ - это каталог по умолчанию, в котором bind9 ожидает хранения файлов зоны
  (по крайней мере, в Debian; я не знаю, следуют ли этому другие дистрибутивы).

  Кроме того, согласно этой документации, pacman (менеджер пакетов, используемый в Arch linux)
  хранит свой кэш пакетов в каталоге /var/cache/pacman/pkg/ и, скорее всего, не ожидает, что его содержимое будет изменено только им самим.

#######################################
# /var/cache : Application cache data #
#######################################

[Назначение:]

/var/cache предназначен для кэшированных данных из приложений.
[*] Такие данные генерируются локально в результате трудоемких операций ввода-вывода или вычислений.

Приложение должно иметь возможность регенерировать или восстанавливать эти данные.
[!] В отличие от /var/spool, кэшированные файлы могут быть удалены без потери данных.
Данные должны оставаться актуальными между вызовами приложения и перезагрузкой системы.

Срок действия файлов, расположенных в каталоге /var/cache, может истекать в зависимости от приложения,
  -> по решению системного администратора
  -> или одновременно.
  
  Приложение ВСЕГДА ДОЛЖНО БЫТЬ СПОСОБНО ВОССТАНОВИТЬСЯ после ручного удаления этих файлов (обычно из-за нехватки места на диске).
  
  Никаких других требований к формату данных в каталогах кэша не предъявляется.
  
[Обоснование:]
  
  Существование отдельного каталога для кэшированных данных позволяет системным администраторам устанавливать политику работы с дисками и резервным копированием,
  отличную от других каталогов в /var.
