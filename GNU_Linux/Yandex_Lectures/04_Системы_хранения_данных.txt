[00:27:50] Типы дополнительной памяти
	- DAS (Direct Attached Storage) - устройство хранения информации подключено напрямую к компьютеру
	- NAS (Network Attached Storage) - подключено через сеть (по сути файловый сервер)
		Компьютеры взаимодействуют с NAS по специальным сетевым протоколам
	- SAN (Storage Area Network) - ориентированная на работу с серверами система хранения данных
		• ATA over Ethernet (AoE)
		• Fibre Channel Protocol (FCP)
		• Fibre Channel over Ethernet (FCoE)
		• iSCSI
		• InfiniBand
	Система хранения SAN может выглядеть, как полка, наполняемая дисками
	Обычно системы NAS и SAN - это двухъюнитовые корпуса (2 "вершка").
	Есть контроллер (встроенный компьютер).
	RAID-контроллеры имеют быстродействующий процессор для выполнения операций с блоками данных
	и Li-ion батарею (энергонезависимость).
	Батарея нужна в связи с тем, что переда записью на диски, контроллер выполняет необходимые операции в оперативной памяти.
	И чтобы данные не пропали в результате сбоя электропитания, нужен независимый источник питания.
	
	HDD vs SSD (с точки зрения удаления/потери информации)
	HDD:
		♦ при перезаписи HDD чаще всего записывает данные на то же самое место, в результате чего при большом числе циклов чтения-записи
		  информация может быть безвозвратно потеряна;
	SSD:
		◘ при перезаписи на диск контроллер распределяет информацию по разным другим ячейкам, чтобы увеличить срок службы диска
		  (ячейки памяти SSD имеют ограниченное число циклов перезаписи).
		◘ когда SSD "умирает", он перестаёт записывать информацию, при этом частично (или полностью) сохраняя возможность чтения
		
[00:44:00] RAID 
	RAID = Redundant Array of Inexpensive Disks
	RAID = Redundant Array of Independent Disks
	• 0 - striping - чередование блоков (↑ производительность; ↓ надёжность, во столько раз, сколько дисков в RAID)
	• 1 - mirroring - зеркалирование (↑ надёжность, ↑ производительность (при распараллеливании чтения))
	• 2 - чередование блоков с кодом Хэмминга (код Хэмминга - код, контроллирующий ошибки, помехоустойчивый код, практически не встречается)
	• 3 - контрольный байт на отдельном диске (встречается в аппаратных контроллерах)
	• 4 - контрольный блок на отдельном диске (контрольная сумма хранится на отдельном диске)
	• 5 - контрольный блок на одном из дисков (с чередованием) (+ надёжность)
	• 6 - 2 контрольных блока (с чередованием) (++ надёжность)

[00:49:40] На чём основаны контрольные суммы? XOR
	- на функции суммирования по модулю 2
	Функция возвращает значение "истина", когда аргументы различны
	0 XOR 0 = 0
	0 XOR 1 = 1
	1 XOR 0 = 1
	1 XOR 1 = 0
	
	Восстановление данных производится путём суммирования двух оставшихся столбцов (1 + результат, 2 + результат)
	(иллюстрации принципов работы разных RAID)

[00:59:32] Нестандартные типы RAID. RAID DP
	- это RAID с двойной чётностью (с диагональной чётностью).
	Разновидность RAID 6.
	Распределение блоков с контрольной суммой немного иное, чем в RAID 6.
	Блоки считаются не по строке, а по диагонали. В результате чего наблюдается прирост производительности.
