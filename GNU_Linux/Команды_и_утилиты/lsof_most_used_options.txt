Наиболее используемые опции
{lsof man page: https://linux.die.net/man/8/lsof
"Утилита lsof - инструмент администратора": https://www.opennet.ru/base/sys/lsof_util.txt.html
}

+d s
    - эта опция заставляет lsof искать все открытые экземпляры каталога s и содержащиеся в нем файлы и каталоги на его верхнем уровне.
	Эта опция НЕ спускается вниз по дереву каталогов, корнем которого является каталог s.
	Опция +D D может быть использована для запроса поиска по дереву каталогов с полным спуском, корнем которого является каталог D. 
Обработка опции
    +d не выполняет поиск по символическим ссылкам внутри s, если только не указана опция -x или -x " l". Также не производится поиск открытых файлов в точках монтирования файловой системы в подкаталогах s, если не указана опция -x или -x " f". 
Примечание: полномочия пользователя этой опции ограничивают ее поиском
    файлов, которые пользователь имеет право исследовать с помощью системной функции stat(2). 
-d s
    - эта опция задает список дескрипторов файлов (FD), которые следует исключить из выходного листинга или включить в него.
	Дескрипторы файлов указываются в наборе s, разделенном запятыми - например, ''cwd,1,3'', ''^6,^2''. (В этом наборе не должно быть пробелов). 
Список является списком исключений, если все элементы набора начинаются с '^'.
    Он является списком включения, если ни одна запись не начинается с '^'. Смешанные списки не допускаются. 
Диапазон номеров дескрипторов файлов может находиться в наборе до тех пор, пока
    ни один из членов не является пустым, оба члена являются числами, и конечный член больше начального - например, ''0-7'' или ''3-10''. Диапазоны могут быть указаны для исключения, если они имеют префикс '^' - например, ''^0-7'' исключает все файловые дескрипторы от 0 до 7. 
Несколько номеров файловых дескрипторов объединяются в один набор ORed, прежде чем
    участия в выборе опции AND. 
Если в наборе есть члены исключения и включения,
    lsof сообщает о них как об ошибках и завершает работу с ненулевым кодом возврата. 
Смотрите описание выходных значений дескриптора файла (FD) в разделе
    OUTPUT для получения дополнительной информации об именах дескрипторов файлов. 
+D D
    Эта опция заставляет lsof искать все открытые экземпляры каталога D и все содержащиеся в нем файлы и каталоги на всю его глубину. 
Обработка опции
    +D не следует за символическими ссылками внутри D, если не указана опция -x или -x " l". Также не производится поиск открытых файлов в точках монтирования файловой системы в подкаталогах D, если не указана опция -x или -x " f". 
Примечание: полномочия пользователя этой опции ограничивают ее поиском
    файлов, которые пользователь имеет право исследовать с помощью системной функции stat(2). 
Дополнительное замечание:
    lsof может обрабатывать эту опцию медленно и требовать для этого большого количества динамической памяти. Это связано с тем, что он должен спуститься по всему дереву каталогов с корнем в D, вызвать stat(2) для каждого файла и каталога, составить список всех найденных файлов и искать в этом списке совпадения с каждым открытым файлом. Если каталог D большой, эти шаги могут занять много времени, поэтому используйте эту опцию с осторожностью.
-D D
    Эта опция направляет использование lsof файла кэша устройств.
	Использование этой опции иногда ограничено.
	Дополнительную информацию об этой опции см. в разделе ФАЙЛ КЭШа УСТРОЙСТВА и следующих за ним разделах.
