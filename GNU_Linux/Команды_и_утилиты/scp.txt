scp (secure copy)
scp - безопасное копирование файлов посредством OpenSSH

Синтаксис команды:
  $ scp [options] source target
    source - источник копируемых файлов (==откуда копируем);
	target - целевой хост, принимающий копируемые файлы (==куда копируем)
	
  $ scp [опции] пользователь1@хост1:файл пользователь2@хост2:файл
  

Некоторые опции:

  -A  = разрешает переадресацию ssh-agent(1) на удаленную систему. 
        По умолчанию агент аутентификации не пересылается

  -B  = выбрать пакетный режим (batch mode, предотвращает запрос паролей или парольных фраз)
  -C  = разрешить сжатие (передает флаг -C в ssh для включения сжатия)
  -c <cipher>  = выбор шифра, используемого для шифрования передачи данных.
        Этот параметр непосредственно передается в ssh.

  -i <identity_file>  = выбрать файл, из которого считывается идентификатор (закрытый ключ) для аутентификации с открытым ключом.
        Эта опция непосредственно передается в ssh

  -o <ssh_option>  = может использоваться для передачи опций в ssh в формате, используемом в ssh_config.
        (удобно для указания опций, для которых не существует отдельного флага командной строки scp.)
		(подробное описание опций ssh_config: https://manpages.ubuntu.com/manpages/lunar/en/man5/ssh_config.5.html)

  -P <port>  = указать порт, к которому необходимо подключиться на удаленном хосте
     #‼ опция записывается с большой буквы 'P'

  -p  = сохранить время модификации, время доступа и биты режима файла из исходного файла.

  -R  = копирование между двумя удаленными хостами осуществляется путем подключения к исходному хосту и выполнения на нем команды scp.
        Для этого необходимо, чтобы scp, запущенный на исходном узле, мог аутентифицироваться на целевом узле, НЕ ТРЕБУЯ ПАРОЛЯ

  -r  = рекурсивное копирование целых каталогов.
        ‼ scp следует за символическими ссылками, встречающимися при обходе дерева.
  

  для передачи данных scp использует ssh;
  применяет ту же аутентификацию и обеспечивает ту же безопасность, что и безопасность, что и при входе в систему.

  scp запрашивает пароли или парольные фразы, если они необходимы для аутентификации.
 
  В качестве source и target может быть указано
    • локальное имя пути,
	• удаленный хост с необязательным путем в виде [user@]host:[path]
	• URI в виде scp://[user@]host[:port][/path].
	
‼  Локальные имена файлов могут быть явно указаны с помощью абсолютных или относительных путей, 
  чтобы scp не воспринимал имена файлов, содержащие ':', как спецификаторы хоста.

  При копировании между двумя удаленными хостами, если используется формат URI, порт на целевом хосте НЕ может быть указан, если используется опция -R.

Примеры:
  1. Простое копирование файла с выбором пользователя на target хосте
    $ scp /home/user/file root@target-srv:/root/
	
  2. Копирование директории
    $ scp -r /home/user/photos root@timeweb:/root/

  3. Передать все файлы из определённой папки
    $ scp -r /home/user/photos/* root@timeweb:/root/

  4. Копирование с сервера на хост
    $ scp root@timeweb:/root/file /home/user/

! путь к директории, в которую вы копируете данные, обязательно должен заканчиваться косой чертой,
иначе произойдет перезапись файлов, и вы потеряете данные, хранящиеся в конечной папке.
