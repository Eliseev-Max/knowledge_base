После установки некоторых пакетов появляется pop-up окно
	"Daemons using outdated libraries"
с вопросом:
	"Which services should be restarted?" и списком служб с чекбоксами слева.
	
Комментарий:
	"Некоторые из этих служб могут потребовать кратковременной остановки для установки нужных вам пакетов.
	Вам действительно следует перезапустить все четыре службы: Выберите все четыре, затем OK."
{source: https://askubuntu.com/questions/1341636/why-ubuntu-server-ask-me-to-restart-services}

В Ubuntu при применении обновлений отдельных служб часто возникает необходимость их перезапуска,
чтобы убедиться, что обновления безопасности действительно применяются и используются.

Большинство скриптов сопровождения делают это автоматически для самой службы.
Однако существует длинная цепочка зависимостей, которая также приводит к необходимости перезапуска.

Например, при установке обновленных пакетов libssl1.1 с устранёнными уязвимостями безопасности,
необходимо перезапустить все процессы, которые связаны с пакетами.
В противном случае содержащая уязвимости libssl1.1 по-прежнему будет загружена во всех процессах и будет использоваться для установления соединений.
Скрипт сопровождения libssl1.1 не может найти и перезапустить все компоненты самостоятельно, поэтому нам нужно что-то другое.

Рекомендуется использовать метапакет ubuntu-server для установки needrestart по умолчанию на Ubuntu Server.
Это приведет к тому, что needrestart будет запускаться в конце каждой транзакции apt и документировать:
	• сервисы, которые необходимо перезапустить, т.к. они ссылаются на обновленные библиотеки;
	• есть ли новое ядро для загрузки;
	• есть ли новый микрокод;
	• есть ли виртуальные машины для перезапуска;
	• есть ли контейнеры, которые нужно перезапустить;
и т.д.

По умолчанию, если нужно перезапустить что-либо, интерфейс debconf выдаст запрос.

Перезапуск некоторых служб (демонов) не разрешен по умолчанию, и это довольно неприятно делать даже при наличии разрешения.
Пример:
	перезапуск systemd-logind → сеансы tty / ssh убиваются →
		→ необходимо восстановить соединение обратно, если вы выполните действие для перезапуска.

Основная цель: гарантировать, что в стабильных релизах можно будет быстрее развернуть больше исправлений безопасности с меньшим временем простоя.
	↑ развёртываний исправлений безопасности ↑  + ↓ время простоя ↓
В конечном итоге лучший способ = перезагрузка!
	+ мы можем убедиться, что все последние библиотеки используются всеми процессами;
	- это может привести к значительному количеству простоев
	
‼ Таким образом, внедрение needrestart - это способ устранить пробел в безопасности.

Если вы считаете, что needrestart слишком мешает вашим рабочим процессам, можно изменить его конфигурацию так, чтобы он не выдавал запросов и выполнял операции только в режиме списка. Если это тоже раздражает, можно remove/uninstall needrestart.
[source: https://discourse.ubuntu.com/t/needrestart-for-servers/21552]
