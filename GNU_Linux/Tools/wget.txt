Утилита wget:
	* предназначена для скачивания файлов и интернет-страниц.
	* работает с протоколами:
		• HTTP
		• HTTPS
		• FTP
	
	* поддерживается работа через прокси
	* может выполнять загрузку файлов в фоновом режиме, без участия пользователя
	* реализует рекурсивную загрузку (функция открытия ссылок на страницы → сохранение целых сайтов)

Синтаксис:
	$ wget [опции] URL [URL2 URL3]
	
Опции:
	• опции можно записывать как до URL, так и после
	• между опцией и её значением не обязательно ставить пробел ( -o log == -olog)
	• если у опции нет параметров (флаги), не обязательно начинать каждую с "-"
	  (-drc == -d -r -c)
	
	Основные опции:
	-V (--version) - вывести версию программы
    -h (--help) - вывести справку
    -b (--background) - работать в фоновом режиме
    -o файл (--out-file) - указать лог файл
    -d (--debug) - включить режим отладки
    -v (--verbose) - выводить максимум информации о работе утилиты
    -q (--quiet) - выводить минимум информации о работе
    -i файл (--input-file) - прочитать URL из файла
    --force-html - читать файл указанный в предыдущем параметре как html
    -t (--tries) - количество попыток подключения к серверу
    -O файл (--output-document) - файл в который будут сохранены полученные данные
    -с (--continue) - продолжить ранее прерванную загрузку
    -S (--server-response) - вывести ответ сервера
    --spider - проверить работоспособность URL
    -T время (--timeout) - таймаут подключения к серверу
    --limit-rate - ограничить скорость загрузки
    -w (--wait) - интервал между запросами
    -Q (--quota) - максимальный размер загрузки
    -4 (--inet4only) - использовать протокол ipv4
    -6 (--inet6only) - использовать протокол ipv6
    -U (--user-agent)- строка USER AGENT отправляемая серверу
    -r (--recursive)- рекурсивная работа утилиты
    -l (--level) - глубина при рекурсивном сканировании
    -k (--convert-links) - конвертировать ссылки в локальные при загрузке страниц
    -P (--directory-prefix) - каталог, в который будут загружаться файлы
    -m (--mirror) - скачать сайт на локальную машину
    -p (--page-requisites) - во время загрузки сайта скачивать все необходимые ресурсы

Использование wget
	(Установка:
		$ sudo apt install wget
		$ yum -y install wget)
	
	Загрузка файла:
		$ wget <URL_файла>
	# wget linux скачает один файл и сохранит его В ТЕКУЩИЙ ДИРЕКТОРИЙ
	# Во время загрузки увидим:
	#	• прогресс загрузки;
	#	• размер файла
	#	• дату последнего изменения файла
	#	• скорость загрузки

	Пример:
	  $ wget http://ftp.gnu.org/gnu/wget/wget-1.5.3.tar.gz
	
	► сохранить файл с другим именем:
	  $ wget -O <new_filename>.zip http://ftp.gnu.org/gnu/wget/wget-1.5.3.tar.gz
	
	► cкачать несколько файлов
	  $ wget http://ftp.gnu.org/gnu/wget/wget-1.5.3.tar.gz ftp://ftp.gnu.org/gnu/wget/wget-1.10.1.tar.gz.sig
	
	► взять URL из файла (файл /wget/tmp.txt)
	  $ whet -i /wget/tmp.txt
	
	► продолжить загрузку (если при загрузке файла было потеряно соединение)
	  $ wget -c http://ftp.gnu.org/gnu/wget/wget-1.5.3.tar.gz
	  # без параметра [-c] загрузка файла начнётся заново
	
	► загрузка файлов в фоне (весь вывод будет записан в лог файл)
	  # за фоновый режим загрузки отвечает параметр [-b, --background]
	  $ wget -b -o ~/wget.log http://ftp.gnu.org/gnu/wget/wget-1.5.3.tar.gz
	# для настройки лог файла используются специальные ключи wget (см. GNU Wget 1.21.1-dirty Manual: 2.4 Logging and Input File options )
	# -o logfile == --output-file=logfile
	
	► ограничение скорости загрузки (до 100 кбит)
	  $ wget --limit-rate=100k ftp://ftp.iinet.net.au/debian/debian-cd/8.4.0/amd64/iso-dvd/debian-8.4.0-amd64-DVD-1.iso
	  # доступны индексы для указания скорости:
	  # • (обычное число) - байты
	  # • k - килобит
	  # • m - мегабит
	  # • g - гигабит
	
	► подключение по логину и паролю (аутентификация на стороне интернет-ресурса)
		# опции для аутентификации:
	  # --http-user=<username>
	  # --http-password=<password>
	  # --ftp-user=<username>
	  # --ftp-password=<password>
	  
	  $  wget --http-user=narad --http-password=password http://mirrors.hns.net.in/centos/6.3/isos/x86_64/CentOS-6.3-x86_64-LiveDVD.iso
	  $ wget --ftp-user=narad --ftp-password=password ftp://ftp.iinet.net.au/debian/debian-cd/6.0.5/i386/iso-dvd/debian-6.0.5-i386-DVD-1.iso
	
	► загрузить и выполнить (для скачанных скриптов)
	  $  wget -O - http://сайт/script.sh | bash
	  # Если опции -O не передать аргументов, то скачанный файл будет выведен в стандартный вывод,
	  # затем мы его можем перенаправить с интерпретатор bash (см.↑)
	
	► сохранить в папке (выбрать папку, отличающуюся от текущей)
	  # опция -P
	  $ wget -P ~/Downloads/ http://ftp.gnu.org/gnu/wget/wget-1.5.3.tar.gz
	
	► передать информацию о браузере (на случай, если сайт фильтрует ботов)
	  # --user-agent=<browser_info> - (user-agent), инфо о браузере
	  # --refer=<from_page> - (http-referer), страница, с котрой мы пришли
	  $  wget --refer=http://google.com --user-agent="Mozilla/5.0 Firefox/4.0.1" //losst.pro
	
	► количество попыток загрузки (по умолчанию 20)
	  # параметр --tries
	  $ wget --tries=75 http://mirror.nbrc.ac.in/centos/7.0.1406/isos/x86_64/CentOS-7.0-1406-x86_64-DVD.iso
		
	► квота загрузки (если количество трафика ограничено)
	  $ wget -Q10m -i download-list.txt		# можем скачать до 10 Мб включительно
	  # индексы те же, что и для указания скорости
	
	► скачать сайт (для просмотра в офлайне)
	  $ wget --mirror -p --convert-links -P ./<Local-Folder> аддресс_сайт
	
	‼ wget поддерживает фильтрацию скачиваемых файлов при их рекурсивном поиске:
		• по суффиксу (~расширение, конечная часть файла) 
		• по шаблону (регулярное выражение, содержащее подстановочные символы: * [...] )
	  Опции для фильтрации:
	    --accept, -A → загрузить файлы, соответствующие указанной маске суффикса\регулярному выражению
		--reject, -R → загрузить файлы КРОМЕ соответствующих указанной маске суффикса\регулярному выражению
	  ! Можно передать в параметр -А  (|-R) несколько суффиксов, разделённых запятыми
	[подробнее - см. GNU Wget 1.21.1-dirty Manual: 4.2 Types of Files]
	
	‼ строки запроса (строки в конце URL, начинающиеся со знака вопроса ('?')) не включаются в имя файла для правил принятия/отклонения,
	хотя они и будут способствовать выбору имени локального файла.
	
	Работа wget с Прокси
		
		Wget поддерживает прокси для получения данных как по HTTP, так и по FTP.
		Указать расположение прокси-сервера:
		• http_proxy = URL
		• https_proxy = URL
		
		♦ --no_proxy = "string"
