Конфигурационные файлы apt
[main source: https://manpages.ubuntu.com/manpages/jammy/man5/apt.conf.5.html]
	
	apt.conf - это основной файл с настройками для инструментов APT.
	Все инструменты используют данный файл с настройками и общий анализатор командной строки для предоставления унифицированного окружения.
	
	Когда инструмент APT запускается, он считывает конфигурационные файлы в следующем порядке:
		1. файл, указанный переменной окружения APT_CONFIG (если таковая имеется);
		2. все файлы в Dir::Etc::Parts в алфавитно-цифровом порядке по возрастанию,
		   которые не имеют или имеют расширение "conf" в качестве имени файла и содержат только:
			    • буквенно-цифровые символы,
				• дефис (-),
				• знак подчеркивания (_)
				• точку (.)
			В противном случае APT выведет уведомление о том, что он проигнорировал файл.
			‼ Если файл соответствует шаблону в списке конфигурации Dir::Ignore-Files-Silently, он будет проигнорирован без вывода уведомления.
		3. главный конфигурационный файл, указанный Dir::Etc::main
		4. все опции, установленные в поддереве конфигурации, специфичной для бинарных файлов, перемещаются в корень дерева.
		5. опции командной строки применяются для отмены директив конфигурации или для загрузки еще большего количества конфигурационных файлов.
	
	
	Файл с настройками организовывается в виде дерева с опциями, распределёнными (организованными) по функциональным группам.
	Спецификация опции задаётся с помощью нотации из "::" (двух двоеточий):
		(например APT::Get::Assume-Yes - это опция внутри группы инструментов APT, для инструмента GET).
	Опции не наследуют от своих родительских групп.
	
	Синтаксически язык конфигурации основывается на инструментах ISC, таких как bind и dhcp.
	
	Комментарии
	Строки, начинающиеся с //, рассматриваются как комментарии (=игнорируются),
	также как и весь текст между /* и */, подобно комментариям в C/C++
	Строки, начинающиеся с #, также рассматриваются как комментарии.

	Каждая строка имеет форму:
		APT::Get::Assume-Yes "true"; 
	
	‼ Кавычки и точка с запятой обязательны.
	Значение должно быть В ОДНОЙ СТРКЕ, конкатенация строк НЕ ДОПУСКАЕТСЯ.
	Значения не должны включать обратные слеши (\) или дополнительные кавычки.
    Имена опций состоят из буквенно-цифровых символов и символов "/-:._+".
		   
	Указанием фигурных скобок можно открыть новый контекст:
		APT {
		  Get {
			Assume-Yes "true";
			Fix-Broken "true";
		  };
		};
	где текст разбит по строкам с добавлением пробелов для удобства чтения.
	Списки можно создавать, открывая область видимости и включая одну строку, заключенную в кавычки, за которой следует точка с запятой.
	Можно включить несколько записей, разделив их точкой с запятой.
		DPkg::Pre-Install-Pkgs {"/usr/sbin/dpkg-preconfigure --apt";};
			равносильно
		dpkg::pre-install-pkgs {"/usr/sbin/dpkg-preconfigure --apt";};
	► ‼ Регистр не имеет значения в именах элементов конфигурации!
	
	В целом, пример конфигурационного файла 
		/usr/share/doc/apt/examples/configure-index.gz
	является хорошим руководством к тому, как он должен выглядеть.
	
	Имена для элементов конфигурации НЕОБЯЗАТЕЛЬНЫ, если определен список (как видно из примера DPkg::Pre-Install-Pkgs выше).
	♦ если не указать имя, то новая запись просто добавит новый параметр в список.
	♦ если указать имя, появится возможность отменить опцию так же, как и любую другую опцию, переназначив ей новое значение.
	
	Две специальные команды:
	  <эти строки также должны заканчиваться точкой с запятой;>
		► #include
			• является устаревшей и не поддерживается альтернативными реализациями
			• включает заданный файл, если только имя файла не заканчивается косой чертой (/), в этом случае включается весь каталог
			
		► #clear
			• используется для стирания части дерева конфигурации (указанный элемент и все его потомки будут удалены)
			• единственный способ удалить список или всю область видимости (scope)
			Повторное открытие области видимости (scope) не отменяет ранее внесённые записи.
	
	◘ Опция -o инструментов APT:
		Все инструменты APT используют опцию -o, которая позволяет указать произвольную конфигурационную директиву в командной строке.
	
		Синтаксис
		(== полное имя опции, за которым следует знак равенства, а затем новое значение опции)
			<полное::имя::опции>=[новое_значение] 
		Пример полоного имени опции - APT::Get::Assume-Yes
		‼ Использование приведённого выше синтаксиса не отменяет ранее записанные записи.
		► Опции могут быть отменены только путем присвоения им нового значения!
		► Списки и области действия не могут быть отменены, только очищены.
		
	Чтобы добавить новый элемент к списку, добавьте к имени списка символ ::
	► (‼) синтаксис scope не может быть использован в командной строке.
	‼ добавление элементов в список с помощью :: работает только для одного элемента в строке,
	и его не следует использовать в сочетании с синтаксисом scope (который добавляет :: неявно).
	Использование обоих синтаксисов вместе приведет к ошибке, от которой, к сожалению, зависят некоторые пользователи:
		опция с необычным именем "::" действует как любая другая опция с таким именем.
	Это создает множество проблем:
		1. пользователи, которые пишут несколько строк в этом неправильном синтаксисе в надежде на добавление к списку, добьются обратного,
		   поскольку будет использовано только последнее назначение для этой опции "::".
		► Будущие версии APT будут выдавать ошибки и прекращать работу, если встретят такое неправильное использование.

Информация о конфигурационных файлах:
	/etc/apt/sources.list
		- места, из которых будут браться пакеты.
		Элемент конфигурации: Dir::Etc::SourceList.
		
	/etc/apt/apt.conf
		- Файл конфигурации APT
		Элемент конфигурации: Dir::Etc::Main.
		
	/etc/apt/apt.conf.d/
		- фрагменты файла конфигурации APT.
		Элемент конфигурации: Dir::Etc::Parts.
		
	/etc/apt/preferences
		- файл предпочтений версии.
		Здесь указывается "привязка", т.е. предпочтение получать определенные пакеты из отдельного источника или из другой версии дистрибутива. 
		Элемент конфигурации: Dir::Etc::Preferences.
		
	/var/cache/apt/archives/
		- область хранения для извлеченных файлов пакетов. 
		Элемент конфигурации: Dir::Cache::Archives.
		
	/var/cache/apt/archives/partial/
		- область хранения для транзитных файлов пакетов. 
		Элемент конфигурации: Dir::Cache::Archives (неявный частичный).
		
	/var/lib/apt/lists/
		- область хранения информации о состоянии каждого ресурса пакета, указанного в sources.list(5). 
		Элемент конфигурации: Dir::State::Lists.
		
	/var/lib/apt/lists/partial/
		- область хранения транзитной информации о состоянии.
		Элемент конфигурации: Dir::State::Lists (неявный частичный).
