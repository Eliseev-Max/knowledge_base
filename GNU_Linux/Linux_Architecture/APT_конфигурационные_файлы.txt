Конфигурационные файлы apt
[main source: https://manpages.ubuntu.com/manpages/jammy/man5/apt.conf.5.html]
	
	apt.conf - это основной файл с настройками для инструментов APT.
	Все инструменты используют данный файл с настройками и общий анализатор командной строки для предоставления унифицированного окружения.
	
	Когда инструмент APT запускается, он считывает конфигурационные файлы в следующем порядке:
		1. файл, указанный переменной окружения APT_CONFIG (если таковая имеется);
		2. все файлы в Dir::Etc::Parts в алфавитно-цифровом порядке по возрастанию,
		   которые не имеют или имеют расширение "conf" в качестве имени файла и содержат только:
			    • буквенно-цифровые символы,
				• дефис (-),
				• знак подчеркивания (_)
				• точку (.)
			В противном случае APT выведет уведомление о том, что он проигнорировал файл.
			‼ Если файл соответствует шаблону в списке конфигурации Dir::Ignore-Files-Silently, он будет проигнорирован без вывода уведомления.
		3. главный конфигурационный файл, указанный Dir::Etc::main
		4. все опции, установленные в поддереве конфигурации, специфичной для бинарных файлов, перемещаются в корень дерева.
		5. опции командной строки применяются для отмены директив конфигурации или для загрузки еще большего количества конфигурационных файлов.
	
	
	Файл с настройками организовывается в виде дерева с опциями, сконцентрированными по функциональным группам.
	Спецификация опции задаётся с помощью нотации из "::" (двух двоеточий):
		(например APT::Get::Assume-Yes - это опция внутри группы инструментов APT, для инструмента GET).
	Опции не наследуют от своих родительских групп.
	
	Синтаксически язык конфигурации основывается на инструментах ISC, таких как bind и dhcp.
	Строки, начинающиеся с //, рассматриваются как комментарии (=игнорируются),
	также как и весь текст между /* и */, подобно комментариям в C/C++
	Строки, начинающиеся с #, также рассматриваются как комментарии.
	Каждая строка имеет форму:
		APT::Get::Assume-Yes "true"; 

	‼ Кавычки и точка с запятой обязательны.
	Значение должно быть В ОДНОЙ СТРКЕ, конкатенация строк НЕ ДОПУСКАЕТСЯ.
	Значения не должны включать обратные слеши (\) или дополнительные кавычки.
    Имена опций состоят из буквенно-цифровых символов и символов "/-:._+".
	   
	Указанием фигурных скобок можно открыть новый контекст:
		APT {
		  Get {
			Assume-Yes "true";
			Fix-Broken "true";
		  };
		};
	где текст разбит по строкам с добавлением пробелов для удобства чтения.
	Списки могут быть созданы путем открытия нового контекста включая одиночное слово, заключенное в кавычки, за которым должна следовать точка с запятой.
	Обычно файл с настройками:
	  /usr/doc/apt/examples/apt.conf
	    и
	  /usr/doc/apt/examples/configure-index
	  (или /usr/share/doc/apt*/examples/configure-index в Conectiva)
	являются хорошим руководством на тему того, как должны выглядеть файлы с настройками.
	
	Опция -o инструментов APT:
		Все инструменты APT понимают опцию -o, которая позволяет указать нужную директиву настройки в командной строке.
	
		Синтаксис 
		<полное::имя::опции>=[новое_значение] 
		Пример полоного имени опции - APT::Get::Assume-Yes
		
	Списки могут быть дополнены также через добавление завершающего "::" к имени списка.
