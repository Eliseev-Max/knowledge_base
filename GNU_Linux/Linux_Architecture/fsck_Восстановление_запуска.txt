1. При попытке загрузки ОС Kubuntu выводится сообщение:

/dev/sda1: Inodes that were part of corrupted orphan linked list found.
/dev/sda1: UNEXPECTED INCONSISTENCY: RUN fsck MANUALLY.
(i. e., without -a or -p options)
[
	/dev/sda1: были найдены inod-ы, являющиеся частью повреждённого связного списка.
	/dev/sda1: НЕОЖИДАННАЯ НЕСОГЛАСОВАННОСТЬ ДАННЫХ: ЗАПУСТИТЕ fsck ВРУЧНУЮ. (т. е. без ключей -a и -p)
]
ОПИСАНИЕ ПРОБЛЕМЫ:
	На /dev/sda1 имеются повреждения файловой системы.
	Единственное, что потеряется при восстановлении — уникальный номер и, как результат, путь до соответствующего файла;
	(он будет помещён в папку lost+found в корне /dev/sda1).
	Главное, чтобы это не оказался какой-нибудь системный файл, ожидаемый во время загрузки ОС по конкретному пути.

ОПРЕДЕЛЕНИЯ И ДОПОЛНЕНИЯ:
(busybox) - командная строка минимальной версии консоли с минимальным набором команд для диагностики и восстановления ОС.

СПОСОБ ВОССТАНОВЛЕНИЯ:
	Общий вид команды:
		$ fsck [опции] [опции_файловой_системы] [раздел_диска]
		
	Запуск утилиты fsck вручную:
		fsck -y /dev/sda1 ; reboot -f
	-y	- отвечает на все вопросы об исправлении ошибок утвердительно
	Пояснение к действиям:
	fsck -y /dev/sda1	- выполнить проверку и безусловное (параметр -y) исправление ошибок;
	reboot -f			- выполнить перезагрузку сразу же по окончании операции.
	
ВНИМАНИЕ!
	fsck -y может убить важные данные и/или всю ФС. Но вероятность этого мала.
	Рекомендация: запустить утилиту без параметра -y и посмотреть, что предлагается исправить.
	ПРОВЕРИТЬ НЖМД на наличие критичных ошибок, возможно, он начал "сыпаться".
