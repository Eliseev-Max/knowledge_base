Зависимости в Ubuntu:
======================
- программу можно установить, *только если уже установлены пакеты, от которых она зависит*.
Такая схема позволяет избежать дублирования данных в пакетах (например, если несколько программ зависят от одной и той же библиотеки, то не придётся пихать эту библиотеку в пакет каждой программы — она поставится один раз отдельным пакетом).
Отличие от Slackware и Windows
Ubuntu зависимости разрешаются пакетным менеджером (Synaptic, apt, Центр приложений, apt-get, aptitude) ­— он !автоматически установит! зависимости из репозитория.
Зависимости придётся устанавливать вручную, если 
* нужный репозиторий не подключен;
* репозиторий недоступен;
* нужного пакета нет в репозитории;
* пакеты устанавливаются без использования пакетного менеджера (используется Gdebi или dpkg)
* программа устанавливается не из пакета (компилируется из исходников, запускается установочный run/sh скрипт).

*************************************************************************
***********************Установка из репозиториев*************************
*************************************************************************
Репозиторий - место централизованного хранения пакетов программного обеспечения.
Список используемых репозиториев содержится в файле /etc/apt/sources.list и в файлах каталога /etc/apt/sources.list.d/

Установка с использованием GUI
Система > Администрирование > Менеджер пакетов Synaptic
Установка с использованием командной строки
sudo apt-get update
sudo apt-get install имя-программы
Если нужно установить несколько программ, то их можно перечислить через пробел.
Для поиска программы в списке доступных пакетов:
+++++++++++++++++++++++++++++++
sudo apt-cache search keyword
+++++++++++++++++++++++++++++++
где keyword - название программы, часть названия программы или слово из её описания
