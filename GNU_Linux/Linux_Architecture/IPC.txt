Межпроцессное взаимодействие (Inter-process communication (IPC)) — это набор методов для обмена данными между потоками процессов.
Процессы могут быть запущены как на одном и том же компьютере, так и на разных, соединенных сетью.

Типы IPC:
	• буфер обмена (clipboard)
	• сокет 
	• сигнал
	• именованный канал (pipe)
	• семафор
	• файл
	• сообщение (WM_COPYDATA)

Каналы (pipes)
	Каналы делятся на анонимные (anonymous pipes) и именованные (named pipes).
	Анонимные каналы:
		► используются достаточно редко;
		► просто передают поток вывода одного процесса на поток ввода другого.
		
	Именованные каналы:
		► передают произвольные данные;
		► могут работать через сеть
		
	(В приведённом примере функция mkfifo() созжаёт специальный FIFO файл со значением umask (правами доступа) = 0777)
	После создания файла любой процесс может открыть его для чтения или записи.
	Однако, для корректного использования файла, необходимо открыть его одновременно двумя процессами/потоками:
	• одним - для получение данных (чтение файла),
	• другим -  на передачу (запись в файл).
	
Разделяемая память (Shared memory)
	Этот способ взаимодействия реализуется не совсем напрямую, а через технологию File Mapping - 
	отображения файлов на оперативную память.
	Механизм позволяет осуществлять доступ к файлу таким образом, как будто это обыкновенный массив,
	хранящийся в памяти (не загружая файл в память явно).
	Работая с этой памятью, потоки обязательно должны согласовывать свои действия с помощью объектов синхронизации.

Удаленный вызов процедур (Remote Procedure Call, RPC)
	Строго говоря, это не совсем технология IPC, а скорее способ значительно расширить возможности других механизмов IPC.
	С помощью этой технологии общение через сеть становится совешенно прозрачным как для сервера, так и для клиента.
	Им обоим начинает казаться, что их "собеседник" расположен локально по отношению к ним.
