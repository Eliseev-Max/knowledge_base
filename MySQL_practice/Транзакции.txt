Если нам необходимо гарантированно выполнить две (или более) команды одновременно
(например, денежный перевод со счёта А на счёт Б: нужно одновременно списать сумму
со счёта А и начислить эту сумму на счёт Б).
Транзакции гарантируют, что будут выполнены все действия, либо не будет выполнено ни одного
из действий.
Пример:
START TRANSACTION;
	UPDATE `shop`.`user_bank_account` SET `money` = `money` - 100 WHERE id =1;
	UPDATE `shop`.`user_bank_account` SET `money` = `money` + 100 WHERE id =2;
COMMIT
 Свойство транзакций ACID
	A (Atomicity) -  Атомарность
	C (Consistency) - Согласованность
	I (Isolation) - Изолированность (Если транзакция началась, её ничто не должно останавливать)
	D (Durability) - Долговечность
