
##########################
# Specify a project name #
##########################
(https://docs.docker.com/compose/project-name/)

  В Compose имя проекта по умолчанию формируется из базового имени каталога проекта.
  Но можно задать собственное имя проекта.

[!] По умолчанию каталог проекта является базовым каталогом файла Compose.
    Для него можно задать и собственное значение с помощью CLI-опции --project-directory.

[!] Compose использует имя проекта для изоляции окружений друг от друга.

  Когда имя проекта может быть полезным:
    * На хосте разработки:
      -> создание нескольких копий одного окружения, полезное для запуска стабильных копий для каждой ветки проекта.

    * На CI-сервере:
      -> Предотвращение помех между сборками путем установки имени проекта на уникальный номер сборки.

    * На общем хосте или хосте разработки:
      -> предотвращение помех между различными проектами, которые могут использовать одни и те же служебные имена.

******************************
* Требования к имени проекта *
******************************
  (!) Имена проектов должны содержать только:
        > строчные буквы,
        > десятичные цифры,
        > тире    и
        > символы подчеркивания
  (!) Имена проектов должны начинаться со строчной буквы или десятичной цифры.

[!]  Если базовое имя каталога проекта или текущего каталога нарушает это ограничение, доступны альтернативные механизмы.

*****************************************
* Порядок приоритета для каждого метода *
*        от высшего к низшему:          *
*****************************************

  1) Установить флаг командной строки `-p`
  2) The COMPOSE_PROJECT_NAME environment variable.
  3) Атрибут верхнего уровня name: в вашем файле Compose.
     Или последнее имя:, если вы указываете несколько файлов Compose в командной строке с флагом -f.
  4) Базовое имя директории проекта, содержащего Compose file.
      Или базовое имя первого файла Compose, если вы указываете несколько файлов Compose в командной строке с флагом -f.
  5) Базовое имя текущего каталога, если файл Compose не указан.

---<(2)>---
"Установка или изменение предопределенных переменных окружения в Docker Compose":
{https://docs.docker.com/compose/environment-variables/envvars/}
  Compose уже поставляется с предопределенными переменными окружения.
  Он также наследует общие переменные окружения Docker CLI, такие как DOCKER_HOST и DOCKER_CONTEXT.
  
   Методы установки и изменения предварительно заданных переменных окружения
     -> в самом Compose файле с помощью атрибута environment
     -> с помощью файла environment
     -> из командной строки
     -> из вашей shell

  COMPOSE_PROJECT_NAME
  Устанавливает имя проекта.
  Это значение вместе с именем сервиса добавляется к имени контейнера при запуске
**********************************************
* <project_name>-<service_name>-<process_ID> *
**********************************************

name: myapp

  foo:
    image: busybox
    command: echo "I'm running ${COMPOSE_PROJECT_NAME}"
