docker tag
(https://docs.docker.com/engine/reference/commandline/tag/)

  - создаёт тег TARGET_IMAGE, который ссылается на SOURCE_IMAGE

Синтаксис команды:
  $ docker tag SOURCE_IMAGE[:TAG] TARGET_IMAGE[:TAG]
  
Описание:
  Полное имя образа имеет следующий формат и компоненты:
    [HOST[:PORT_NUMBER]/]PATH
,где
  • HOST
    - необязательное имя хоста реестра определяет, где находится образ.
	  Имя хоста должно соответствовать стандартным правилам DNS, но НЕ ДОЛЖНО СОДЕРЖАТЬ символов подчеркивания.
	  Если имя хоста не указано, команда по умолчанию использует публичный реестр Docker по адресу:
---
registry-1.docker.io
---
[!]  Обратите внимание:
     docker.io - это каноническая ссылка на публичный реестр Docker.

  • PORT_NUMBER
    - если присутствует имя хоста, то за ним по желанию может следовать номер порта реестра в формате :8080

  • PATH
    - Путь состоит из компонентов, разделенных слэшем.
	  Каждый компонент может содержать строчные буквы, цифры и разделители.
	  Допустимые разделители определяются как:
	    -> точка 
		-> одно или два знака подчеркивания
		-> один или несколько дефисов.

	  Компонент НЕ МОЖЕТ НАЧИНАТЬСЯ ИЛИ ЗАКАНЧИВАТЬСЯ РАЗДЕЛИТЕЛЕМ.
	  Хотя спецификация OCI Distribution Specification поддерживает более двух компонентов, разделенных косой чертой,
	  большинство реестров поддерживают только два компонента, разделенных косой чертой.
	  Для публичного реестра Docker формат пути выглядит следующим образом:
	    [NAMESPACE/]REPOSITORY
		- первый, необязательный компонент, как правило, представляет собой пространство имен пользователя или организации.
		- второй, обязательный компонент - это имя репозитория.
		Если пространство имен отсутствует, Docker использует библиотеку в качестве пространства имен по умолчанию.

  После имени образа указывается необязательный тег TAG - пользовательский, человекочитаемый идентификатор манифеста,
  который обычно представляет собой конкретную версию или вариант образа.
  Тег должен быть корректным ASCII и может содержать 
    • строчные и прописные буквы,
	• цифры,
	• подчеркивания,
	• точки,
	• дефисы.
	
  ‼ Тэг не может начинаться с точки или дефиса 
  ‼ Тэг должен содержать не более 128 символов.
  
  Если тег не указан, то по умолчанию используется команда latest.

С помощью имен и тегов можно группировать изображения, а затем помещать их в реестр.

Примеры использования команды docker tag
  
1. Tag an image referenced by ID

    $ docker tag 0e5574283393 fedora/httpd:version1.0

2. Tag an image referenced by Name

    $ docker tag httpd fedora/httpd:version1.0

3. Tag an image referenced by Name and Tag

    $ docker tag httpd:test fedora/httpd:version1.0.test

4. Tag an image for a private registry

    $ docker tag 0e5574283393 myregistryhost:5000/fedora/httpd:version1.0
