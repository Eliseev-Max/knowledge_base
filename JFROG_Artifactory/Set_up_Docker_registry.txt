Пошаговая инструкция насройки репозиториев Docker в Artifactory
[from: How to set up a Private, Remote and Virtual Docker Registry
	https://jfrog.com/blog/how-to-set-up-a-private-remote-and-virtual-docker-registry/
]

Предварительная подготовка:
	1. Установлен Docker
	2. Установлен JFrog Container Registry (self hosted installation)

1. Войти в свою среду
	• Войдите в систему, используя имя пользователя по умолчанию: admin и пароль: password для локальной установки

2. Добавление локального репозитория Docker
   (Используется для хранения пользовательских образов Docker)
	• перейти в модуль администрирования;
	• развернуть меню Repositories и нажать на пункт меню Repositories;
	• добавить новый локальный репозиторий с типом пакета Docker;
	• ввести ключ репозитория (Repository Key), например, "docker-dev-local"; 
	• сохранить остальные настройки по умолчанию.

3. Добавление удалённого репозитория Docker
   (Используется как кэширующий прокси, для хранения сторонних образов из Docker Hub или любых других внешних реестров)
	• перейти на вкладку Remote на странице Repositories;
	• добавить новый удаленный репозиторий с типом пакета Docker;
	• ввести ключ репозитория (Repository Key) "docker-hub-remote"; 
	• сохранить остальные настройки по умолчанию;
	• добавить удаленный репозиторий Docker
	[*1] Примечание

4. Добавление виртуального репозитория Docker
   (Используется при создании пользовательского образа Docker)
	• перейти на вкладку Virtual на странице Repositories;
	• добавить новый виртуальный репозиторий с типом пакета Docker;
	• Ввести ключ репозитория "docker"
	• добавить локальный и удаленный репозитории docker 
	    [(созданы в шагах 2 и 3): переместить их из списка Available Repositories в Selected Repositories с помощью кнопок со стрелками: << | >> ]
		‼ Порядок этих репозиториев в списке будет определять порядок, используемый для разрешения зависимостей,
		необходимых для сборки вашего образа docker.
	• выбрать локальный репозиторий, который мы создали в шаге 2, в качестве репозитория развертывания по умолчанию.
		‼ Default Deployment Repository - это хранилище, в которое будет загружен созданный вами образ докера.
	• Сохраните остальные настройки по умолчанию

5. Форк (ветвление) примера JFrog Docker на репозитории GitHub
   (Этот репозиторий содержит простой Dockerfile, который мы используем, чтобы сбилдить свой пользовательский образ)
   
6. Обновление ссылки на базовый образ
	Клонируйте ваш форкнутый репозиторий и обновите строку FROM в Dockerfile,
	чтобы она ссылалась на ваш виртуальный репозиторий Docker:
	  FROM ${SERVER_NAME}.jfrog.io/${VIRTUAL_REPO_NAME}/alpine:3.11.5
	  
	    SERVER_NAME - это первая часть URL-адреса, предоставленного вам для вашей среды: https://SERVER_NAME.jfrog.io.
	    VIRTUAL_REPO_NAME - это имя "docker",которое вы присвоили своему виртуальному хранилищу в Шаге 4
		

*1  Если не указать учётные данные учётной записи DockerHub, этот ресурс будет проксироваться анонимно, 
	и будет применено ограничение анонимного использования DockerHub.
	Предупреждение, которое выводится, если не вводить учётные данные Docker при создании удалённого репозитория в Artifactory:
	"Docker Hub will be proxied anonymously as you did not set your Docker Account credentials.
	To set the usage limit, enter your Docker credentials or confirm using Docker Hub anonymously."
	
	Компания Docker наложила ограничения на анонимное использование удалённого репозитория Docker (DockerHub):
	анонимные пользователи блокируются при достижении ограничения скорости загрузки в 100 обращений за 6 часов.
	Чтобы этого не произошло, рекомендуется:
		1. пройти процесс аутентификации в DockerHub;
		2. Задать имя пользователя и пароль учётной записи Docker в разделе "Удалённые репозитории Docker".
